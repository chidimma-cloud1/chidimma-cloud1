# History of Cloud Computing

## Overview
Cloud computing didn't start with AWS. It evolved over seven decades, shaped by the need to make computing more accessible, efficient, and affordable. Understanding its history helps you appreciate why the cloud works the way it does today.

## What is Cloud Computing?
Cloud computing is the on-demand delivery of computing services including servers, storage, databases, networking, and software over the internet. Instead of owning and maintaining physical data centers, users rent access to these resources from providers on a pay-as-you-go basis.

## Timeline

### 1950s-1960s: Time-Sharing Mainframes
Computers back then were room-sized machines that cost millions of dollars. Only large institutions governments, universities, and big corporations could afford them. Since nobody could buy a machine for every employee, institutions let multiple users access one machine through basic terminals. This was called time-sharing running multiple users' tasks on one computer at once.

In 1961, John McCarthy the pioneer who coined the term "Artificial Intelligence" predicted at MIT that computing would one day be delivered like a public utility, just like water or electricity. You plug in, use it, and pay only for what you consume. That prediction is exactly how modern cloud billing works today.

### 1970s-1980s: Virtualization Emerges
IBM created VM/370, a system that could run multiple operating systems on a single piece of hardware by splitting it into separate, secure spaces for different users. This concept known as virtualization is the direct technical ancestor of cloud infrastructure today. Every virtual server you spin up on AWS traces back to this idea.

### 1990s: The Internet Changes Everything
As the internet went mainstream, businesses started imagining software delivered over networks instead of installed locally. Engineers even started using a literal cloud symbol in network diagrams to represent "servers somewhere out there that you don't need to think about." This is where the term cloud computing began to take shape. Distributed computing, client-server models, and early hosted services started gaining traction during this era.

### 1999: Salesforce Proves SaaS Works
Salesforce did something radical they delivered a full business CRM application entirely through a web browser with zero local installations or heavy hardware requirements. This proved that Software as a Service (SaaS) was a viable business model and became the blueprint for how software is delivered today.

### 2002-2006: AWS Changes the Game
Amazon launched AWS in 2002 as internal developer tools. Then in 2006, they launched EC2 (Elastic Compute Cloud) and S3 (Simple Storage Service) letting anyone rent virtual servers and store massive amounts of data on demand. Amazon had realized they had massive unused server capacity outside of peak holiday seasons, so they decided to rent it out. This is widely seen as the moment modern cloud computing truly took shape.

### 2008-2010: The Big Players Join In
Microsoft Azure and Google Cloud entered the race, expanding the cloud beyond just storage and computing into databases, analytics, and eventually machine learning. This competition drove innovation and made cloud services more affordable and accessible for businesses of all sizes.

## Why It Matters
Understanding this history shows that the cloud isn't a single invention it's a continuous cycle of centralizing and decentralizing computing power to make technology more efficient, accessible, and scalable. Every tool you use on AWS today is built on decades of innovation.

## Key Takeaway
It all started with people trying to share one expensive computer in the 1960s. Today, anyone with an internet connection can access the same computing power as a Fortune 500 company.
